# EmbedThis Updater - Release 1.2.0

**Release Date:** February 20, 2026
**Tag:** v1.2.0

## Overview

Version 1.2.0 migrates the build system from MakeMe to premake5, aligning the updater with the rest of the EmbedThis module ecosystem. No changes to updater logic or test infrastructure.

## Highlights

- **Build system migration** from MakeMe to premake5
- **Generated project files** for gmake2, Visual Studio 2022, and Xcode
- **Amalgamated distribution** via `bin/buildLib.sh` producing `dist/updaterLib.c`
- **ME_COM_ defaults** added to `src/updater.h` with `#ifndef` guards

## Changes

### Build System (DEV)
- Replaced MakeMe-generated platform makefiles with premake5
- Created `projects/premake5.lua` build configuration
- Generated `projects/gmake2/`, `projects/vs2022/`, `projects/xcode/` project files
- Replaced top-level Makefile to delegate to premake gmake2
- Created `bin/buildLib.sh` for amalgamated `dist/updaterLib.c`
- Deleted all MakeMe artifacts: `main.me`, `start.me`, `make.bat`, `src/me.h`, `src/Makefile`, `bin/fixProjects`, old platform makefiles and configs, old VS/Xcode projects, GUID files, nmake helpers

### Header Changes (DEV)
- Added ME_COM_ defaults to `src/updater.h`: ME_COM_OPENSSL, ME_COM_MBEDTLS, ME_COM_OSDEP, ME_COM_SSL, ME_COM_UPDATER
- Added ME_NAME and ME_TITLE defines
- Wrapped OpenSSL includes in `#if ME_COM_OPENSSL` conditional

### Documentation (DOC)
- Updated README with new build instructions, make targets, and file listing
- Removed references to MakeMe from README
- Added premake5 project regeneration instructions

## Breaking Changes

None. The build output is identical. Users building from source use `make` as before.

## Platform Support

- **Linux** (x64, ARM)
- **macOS** (x64, ARM64/M1)
- **Windows** (x64) via Visual Studio 2022
- **FreeBSD**

## Dependencies

- **OpenSSL** 1.1.1 or later (3.x recommended)
- **C Compiler**: GCC 7+, Clang 10+, or MSVC 2022+
- **TestMe**: For running unit tests
- **premake5**: Only for regenerating project files (not required for building)

## Migration Guide

Users upgrading from 1.1.x can use this release as a drop-in replacement. The `make` command works identically. Windows users should now open `projects/vs2022/updater.sln` instead of the old solution file.
